A Database Management System (DBMS) is a software system designed to manage and organize data in a structured manner. It allows users to create, modify, and query a database, as well as manage security and access controls.

Features of DBMS:

  • Data Modeling: A DBMS provides tools for creating and modifying data models, defining the structure and relationships of the data in a database.
  • Data Storage and Retrieval: A DBMS is responsible for storing and retrieving data from the database, providing various methods for searching and querying the data.
  • Concurrency Control: A DBMS provides mechanisms for controlling concurrent access to the database, ensuring that multiple users can access the data without conflict.
  • Data Integrity and Security: A DBMS enforces data integrity and security constraints, such as constraints on data values and access controls.
  • Backup and Recovery: A DBMS provides mechanisms for backing up and recovering data in the event of system failure.

DBMS can be classified into two types:
RDBMS: Data is organized in tables with rows and columns, related through primary and foreign keys.
NoSQL: Data is organized in key-value pairs, documents, graphs, or column-based structures, designed for large-scale, high-performance scenarios.

A database is a collection of interrelated data organized in tables, views, schemas, etc., facilitating efficient retrieval, insertion, and deletion of data. For example, a university database organizes data about students, faculty, admin staff, etc., facilitating efficient data operations.

Applications of DBMS:

  • Enterprise Information: Sales, accounting, human resources, Manufacturing, online retailers.
  • Banking and Finance Sector: Banks maintaining customer details, accounts, loans, banking transactions, credit card transactions.
  • Finance: Storing information about sales and holdings, purchasing financial stocks and bonds.
  • University: Maintaining information about student course enrolled information, student grades, staff roles.
  • Airlines: Reservations and schedules.
  • Telecommunications: Prepaid, postpaid bills maintenance.

Paradigm Shift from File System to DBMS:
File System manages data using files on a hard disk. Users can create, delete, and update files according to their requirements. However, there are several issues with this system:

  • Redundancy of data
  • Inconsistency of Data
  • Difficult Data Access
  • Unauthorized Access
  • No Concurrent Access
  • No Backup and Recovery

Advantages of DBMS:

  • Data organization
  • Data integrity
  • Concurrent access
  • Data security
  • Backup and recovery
  • Data sharing

Disadvantages of DBMS:

  • Complexity
  • Performance overhead
  • Scalability
  • Cost
  • Limited Use Cases
These are the main reasons which made a shift from the file system to DBMS.

Several Types of DBMS:

  • Relational DBMS (RDBMS)
  • Object-Oriented DBMS (OODBMS)
  • NoSQL DBMS
